The Lambda Expressions, included in C# 3.0, are anonymous functions that can contain expressions and instructions and can be used to create delegates or expression tree types. These provide us the ability to compile them as expression trees, ie data objects in memory representing the algorithm for evaluating the lambda expression itself.

Let’s see a little example:

	    class Program
      {
          static Func makeSomething = (x, y) => Math.Sqrt(x + y);
          static Expression> makeSomethingExpression = (x, y) => Math.Sqrt(x + y);
      
          static void Main(string[] args)
          {
              Console.WriteLine(makeSomethingExpression);
              
              // Generamos ILAsm
              makeSomething = makeSomethingExpression.Compile();
              
              Console.WriteLine("makeSomething(12, 4) = " + makeSomething(12, 4));
              
              Console.ReadLine();
          }
      }

This code is generated for the object tree that represents the sequence of actions necessary for the evaluation of the tree structure generated. Note that the Expression class provides a method Compile(), with which we can dynamically generate executable code for that expression when we want.
